# nerve-wml v1.8.0 — axioms integration

**Date**: 2026-04-24
**Tag**: `v1.8.0`
**Upstream pin**: `dreamofkiki @ v0.9.1` (formal axis `C-v0.8.0+PARTIAL`)

## What changed

- New optional extras group `nerve-wml[axioms]` that installs
  `dreamofkiki` from the `hypneum-lab/dream-of-kiki@v0.9.1` git tag.
  The upstream package is not published to PyPI; the VCS URL is the
  canonical install path until that changes.
- `nerve_core/axioms_compat.py` pins the upstream formal axis version
  (`C-v0.8.0+PARTIAL`) and emits `UpstreamAxiomsVersionWarning` on
  drift. The check runs once at `nerve_core` import; it is a silent
  no-op when the extras group is not installed.
- `nerve_core.from_dream_of_kiki.from_dream_of_kiki` now consumes the
  DR-2 weakened predicate (`_dr2_precondition`) from the upstream
  amendment dated 2026-04-21. If the input spec carries an
  `operation_order` hint AND the DR-2 value is an `Axiom` instance
  with a callable `predicate`, the factory raises
  `DreamOfKikiAxiomError` for orders that violate the precondition
  (RESTRUCTURE before REPLAY).
- The dict-based spec contract that shipped in v1.7.0 is preserved
  unchanged — the new path is strictly additive. Consumers may mix
  `Axiom` instances and plain dicts in the same spec.

## What did NOT change

- `bridge/dream_bridge.py`, `bridge/dream_protocol.py`,
  `bridge/mock_consolidator.py` — consolidate runtime scaffold
  intact. Upstream does not publish `kiki_oniric.consolidate()` yet;
  that integration is v1.9.0.
- Paper artifacts — no re-run, no figure regeneration.
